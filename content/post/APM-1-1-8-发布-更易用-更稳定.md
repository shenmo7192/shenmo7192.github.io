APM 是星火推出的容器软件包管理系统，以简单易用，生态与Debian兼容为特色。

apm 1.1.8 发布，主要更新了两个功能

1. apm launch 包名

APM的命令语义一直和玲珑不太一样，玲珑的 ll-cli run 会启动配置好的entries，ll-cli --exec 则是启动容器内的指定路径的文件，但apm run 和 ll-cli --exec 是一致的，且缺少对应 ll-cli run 的指令，所以命令行启动应用的时候会比较麻烦

现在添加了 apm launch 指令，现在可以和玲珑一样输入包名启动应用了，且支持加参数

比如 apm launch wps-office 后面可以直接加文件路径，直接打开文件

![录屏_gxde-terminal_20260315141508.gif](https://storage.deepin.org/thread/202603150615277477_录屏_gxde-terminal_20260315141508.gif)

此外，为了使用方便，特殊对只有包名的 apm run 参数做了路由，所以输入 apm run wps-office 也是能启动 WPS 的。但为了不影响语义，不支持后续加参数

![录屏_gxde-terminal_20260315141624.gif](https://storage.deepin.org/thread/202603150616434606_录屏_gxde-terminal_20260315141624.gif)

1. 修复在安装失败后卡住dpkg无法继续使用的问题

1. 优化了 bash 自动补全的体验

1. 修复了在星火应用商店安装APM版软件后无法创建.desktop文件的问题

适用于 Debian / deepin / openKylin 的软件包已推送，使用 sudo apt update 即可收到更新

适用于 Arch Linux 的软件包预计今晚推送，使用paru可跟随更新

适用于 Fedora 的软件包仍在等待制作中
